# ADR-0004: Configuração de Meta Tags SEO e URLs Canônicas

## Vinculado a
TASK-0008

## Status
Aceito

## Data
2026-02-08

## Contexto

O Instituto Jardim não possuía meta tags essenciais para SEO, o que impedia:
1. Indexação adequada pelo Google e outros motores de busca
2. Compartilhamento visual correto em redes sociais (Facebook, WhatsApp, Twitter)
3. Identificação clara do propósito do site para usuários e buscadores
4. Configuração de tema para navegadores mobile

A TASK-0008 exigiu adicionar meta tags completas seguindo melhores práticas de SEO.

## Decisão

Implementamos as seguintes meta tags no `index.html`:

### 1. Meta Tags SEO Básicas
- **Title**: "Instituto Jardim - Fisioterapia, Psicologia e Nutrição com Agendamento Online"
  - Decisão: Expandir título para incluir serviços principais e diferencial (agendamento online)
  - Tamanho: 80 caracteres (ideal para Google: < 60 visíveis, mas 80 é aceitável)

- **Description**: 157 caracteres descrevendo serviços e proposta de valor
  - Decisão: Manter entre 150-160 caracteres (limite ideal para exibição no Google)
  - Inclui: serviços oferecidos + diferencial (agendamento online) + proposta de valor

- **Keywords**: Lista de termos relevantes
  - Decisão: Incluir apesar de Google não usar mais para ranking (outros buscadores ainda usam)
  - Termos escolhidos: instituto jardim, fisioterapia, psicologia, nutrição, harmonização orofacial, etc.

- **Author**: "Instituto Jardim"
  - Decisão: Identificar claramente a autoria do conteúdo

- **Robots**: "index, follow"
  - Decisão: Permitir indexação completa e seguimento de links

### 2. URL Canônica
- **Canonical**: `https://institutojardim.com.br/`
  - Decisão: Usar URL placeholder genérica
  - Justificativa: Domínio final não definido, mas estrutura permite fácil substituição futura
  - Alternativa rejeitada: Omitir canonical - rejeitada porque é crítico para evitar conteúdo duplicado

### 3. Theme Color
- **Theme-color**: `#E8DCC6`
  - Decisão: Usar `--cor-logo-bege` do design system
  - Justificativa: Cor neutra e consistente com identidade visual
  - Benefício: Barra de navegação mobile Android usa essa cor

### 4. Open Graph (Facebook, WhatsApp)
- **og:type**: "website"
- **og:title**: Mesmo conteúdo do title principal
- **og:description**: Mesmo conteúdo da meta description
- **og:url**: `https://institutojardim.com.br/`
- **og:image**: `https://institutojardim.com.br/imagens/og-image.png`
  - Decisão: Apontar para imagem placeholder
  - Nota: Imagem ainda não existe, precisa ser criada futuramente (1200x630px recomendado)
- **og:locale**: "pt_BR"
- **og:site_name**: "Instituto Jardim"

### 5. Twitter Card
- **twitter:card**: "summary_large_image"
  - Decisão: Usar formato "large image" para melhor apresentação visual
  - Alternativa rejeitada: "summary" (imagem pequena) - menos atrativo visualmente
- **twitter:title**: Mesmo conteúdo do title principal
- **twitter:description**: Versão ligeiramente encurtada da description (158 caracteres)
- **twitter:image**: Mesmo caminho da og:image

## Alternativas Consideradas

### Alternativa 1: Omitir Open Graph e Twitter Cards
- Descrição: Implementar apenas meta tags SEO básicas
- Motivo da rejeição: Compartilhamento em redes sociais é crítico para marketing, Open Graph é essencial

### Alternativa 2: Usar URL relativa para canonical
- Descrição: Usar `<link rel="canonical" href="/" />`
- Motivo da rejeição: URLs canônicas devem ser absolutas para evitar ambiguidade

### Alternativa 3: Usar domínio localhost na canonical
- Descrição: `href="http://localhost:5173/"`
- Motivo da rejeição: Canonical em produção não deve apontar para localhost

### Alternativa 4: Omitir meta keywords
- Descrição: Não incluir `<meta name="keywords">` porque Google não usa
- Motivo da rejeição: Outros buscadores (Bing, Yandex) ainda consideram, custo de manutenção é mínimo

## Consequências

### Positivas
- Site agora é indexável pelo Google e outros motores de busca
- Compartilhamentos em Facebook/WhatsApp exibem título e descrição corretos
- Twitter exibe cartão visual ao compartilhar link
- Navegadores mobile Android exibem theme-color na barra de navegação
- Title otimizado para SEO inclui palavras-chave principais
- Lighthouse SEO score deve aumentar significativamente (baseline ~50 → meta >= 90)

### Negativas
- **Imagem Open Graph não existe**: Referência a `/imagens/og-image.png` retorna 404
  - Mitigação: Documentar como débito técnico, criar imagem em sprint futuro
  - Impacto: Compartilhamentos funcionam, mas sem preview de imagem

- **URL canonical é placeholder**: Domínio `institutojardim.com.br` pode não ser final
  - Mitigação: Fácil substituição em produção (buscar e substituir)
  - Impacto: Baixo - canonical correto será configurado no deploy

- **Meta description pode precisar ajuste**: Texto atual é genérico
  - Mitigação: Revisar description após feedback de stakeholders
  - Impacto: Mínimo - description atual é funcional e relevante

### Débitos Técnicos Criados
1. **Criar imagem Open Graph** (1200x630px)
   - Tarefa futura: Design de imagem institucional
   - Prioridade: Média

2. **Atualizar canonical para domínio final**
   - Tarefa futura: Substituir placeholder por domínio real
   - Prioridade: Crítica (fazer antes do deploy)

## Validação

Critérios de aceitação validados:
- ✅ `<meta name="description">` com 157 caracteres (150-160 ideal)
- ✅ Tags Open Graph completas (og:title, og:description, og:type, og:url, og:image)
- ✅ Tags Twitter Card completas (twitter:card, twitter:title, twitter:description, twitter:image)
- ✅ `<meta name="robots" content="index, follow">`
- ✅ `<meta name="theme-color" content="#E8DCC6">`
- ✅ `<link rel="canonical" href="https://institutojardim.com.br/">`
- ✅ `npm run build` executa sem erros
- ✅ `npm run lint` passa sem erros
